## 🎬 录制技巧

### 画面稳定性

1. **固定镜头**: 使用固定机位，避免频繁晃动
2. **平滑过渡**: 界面切换时使用淡入淡出效果
3. **焦点清晰**: 确保演示界面始终在焦点内

### 节奏控制

1. **分段录制**: 每个功能点单独录制，后期拼接
2. **留白时间**: 操作后停顿 1-2 秒，让观众看清
3. **适时解说**: 避免操作和解说同时进行

### 声音控制

1. **语速适中**: 保持稳定的说话速度
2. **停顿自然**: 在关键信息前后停顿
3. **语气积极**: 使用自信、热情的语气

## ⚠️ 常见问题处理

### 问题 1: API 响应慢

**应对方案**:
- 提前预热：演示前先分析一次关键词
- 降低数据量：将 `limit_per_platform` 设置为 20
- 准备备用方案：录制一个快速响应的版本

### 问题 2: 采集失败

**应对方案**:
- 准备 Mock 数据：使用本地模拟数据演示
- 说明情况：在解说中说明"网络问题时可能失败"
- 提前测试：演示前测试所有功能

### 问题 3: 界面卡顿

**应对方案**:
- 关闭其他应用：释放系统资源
- 降低复杂度：减少同时展示的数据量
- 剪辑优化：后期加速或跳过卡顿部分
